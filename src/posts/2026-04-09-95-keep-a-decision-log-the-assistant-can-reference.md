Pourquoi as-tu choisi cette base de données plutôt que les alternatives? Pourquoi la couche d'authentification est-elle structurée de cette façon? Pourquoi ce module a-t-il cette interface plutôt que la plus évidente? Si ces décisions vivent seulement dans ta tête, tu vas les re-litiger — avec toi-même, avec ton équipe, et avec l'assistant, qui va proposer les alternatives que tu as déjà rejetées chaque fois qu'il rencontre le point de décision.

Un journal de décisions est un document léger qui enregistre les décisions techniques significatives, les alternatives qui ont été considérées, et le raisonnement derrière le choix fait. Il n'a pas besoin d'être élaboré — quelques phrases par décision suffisent pour fournir le contexte qui empêche la question d'être rouverte improductivement.

La valeur pour l'assistant est spécifique: quand tu inclus le journal de décisions dans le contexte, l'assistant arrête de suggérer les approches que tu as déjà écartées. Il bâtit sur les décisions plutôt que de les questionner. Les conversations deviennent plus productives parce qu'elles commencent à partir de l'état actuel plutôt que de relitiger le chemin qui t'y a amené.

La valeur pour l'équipe est plus large. Les décisions documentées avec le raisonnement sont des décisions qui peuvent être revisitées délibérément quand les circonstances changent, plutôt que défaites accidentellement quand quelqu'un ne sait pas qu'elles ont été prises. Le journal est aussi une mémoire institutionnelle — quand la personne qui a pris la décision part, le raisonnement reste.

Écris les décisions qui ont été difficiles à prendre. Ce sont celles qui seront questionnées à nouveau.
